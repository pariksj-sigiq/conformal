What SFS actually needs after the demo

The demo proved the interaction model. Production needs source access, semantic data, permissions, evals, auditability, deployment, and business ownership. This workbench breaks the 21 use cases into buildable waves.

Recommended wave structure

Anchor on what Conformal can actually own: productionizing the executive assistant, creating the reusable AI/data foundation, then expanding into field force and procurement.

Hard truth

Do not estimate 21 builds blindly. First prove the data and operating model.

The demo is a vertical slice

It proves the user experience and agent reasoning loop, not production readiness.

The platform is not optional

Every real wave needs data access, semantic definitions, identity, audit, and evals.

Logistics is not our custom build

That group is vendor selection, integration, and fit-gap. The AI value is thin.

What we need to learn first

The critical learning is not SAP theory; it is how SFS exposes business data and who owns each definition.

SAP access pattern

CDS/OData/API, BW/Power BI, batch extracts, or file drops. This decides timeline.

Business ownership

Finance, Sales, Procurement, IT, and Supply Chain must name owners for definitions and signoff.

Security envelope

Azure/VPC, SSO, data egress, model provider, audit logs, retention, and legal review.

Use-case inspector

Each case is classified by what it actually requires: agent product, data product, ML model, buy-and-integrate, or foundation.

What you need to learn

Learn enough to ask sharp questions and architect around enterprise constraints. You do not need to become an SAP consultant.

What we need from SFS and from ourselves

The largest risk is not model quality. It is missing owners, missing access, and unclear definitions.
