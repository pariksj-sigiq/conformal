Integration path into a columnar AI store

The clean path is to avoid direct agent access to source systems. Land read-only extracts from SAP BW / Power BI and internal apps, standardize them, create gold marts, then expose governed semantic views to agents.

1

SFS source systems

SAP BW/Power BI, SAP SD/MM/FI, Growthbook, Saathi, Ariba, Concur, Complinity, schemes, logistics, external data.

2

Raw landing

Read-only batch/API/SFTP extracts stored exactly as received with source and run metadata.

3

Columnar standard layer

Typed Parquet/Delta tables, ID crosswalks, quality checks, refresh timestamps.

4

Gold semantic marts

Dealer 360, field performance, spend cube, stock visibility, enterprise KPIs.

5

AI agent tools

SQL over governed views, retrieval over docs, charting, report generation, audit logs.

6

SFS apps

Leadership cockpit, field-force assistant, procurement copilot, alerts and exports.

Cross-cutting net-new capture dependency

SFS states secondary sales, inventory liquidation, and PoG trackers are not collected consistently. That does not block descriptive dashboards, but it blocks the smarter liquidation and channel-stock layer.

Dependency: secondary sales / inventory liquidation / PoG capture.
Affected use cases: Field Force #1 Field Performance, Field Force #2 Activity Analytics, Field Force #6 Product & Partner Visibility, and Channel Stock Estimate fields.
Honest caveat: the descriptive base is fast if SAP and Growthbook data lands; liquidation completeness and PoG-led intelligence need a new SFS capture workflow or distributor/dealer reporting process.

SFS ROI and effort anchors

The data-readiness result converges with SFS's own impact and effort lines. This is why Wave 1 should be ROI-aligned, not just technically convenient.

Field ForceSFS estimates INR 50-80 Cr incremental revenue and an 8-month build. Field Performance is also our fastest data start, so it should anchor Wave 1.
Enterprise AssistantSFS marks it as the shortest 4-month build. We already have the strongest pilot surface if scoped to confirmed SAP BW / Power BI metrics.
ProcurementSFS estimates INR 10-15 Cr savings. Historical Spend can enter early discovery because the SAP purchase-register base is strong.
LogisticsSFS's low custom-build estimate supports buy-and-customize. Keep it as vendor/TMS fit-gap, not a custom agent build.
Recommended Wave 1: platform foundation + Enterprise Assistant + Field Force #1 Field Performance, with Field Force #4 Dealer Profiles as the field-facing companion and Procurement #1 Historical Spend in discovery.

Use-case readiness workbench

Start with the easy and strong-pilot candidates. Keep external-data and new-capture use cases in discovery until SFS validates owners, providers, and workflows.

What to ask SFS next

The goal of the first technical working session is not production integration. It is enough source truth to classify use cases as extraction-led, integration-led, or workshop-heavy.

  1. Confirm source owners for SAP SD, SAP MM, SAP FI/CO, stock/inventory, Growthbook, Saathi, Ariba, schemes, logistics, and Azure/security.
  2. Share masked sample extracts first: SAP purchase register, sales/billing, collections, dealer master, product master, vendor master, targets, field activity, stock snapshots.
  3. Confirm read-only extract access off the existing SAP BW / Power BI layer, including refresh frequency and preserved join keys.
  4. Confirm join keys: dealer ID, material code, vendor ID, employee ID, territory, plant, warehouse, HSN.
  5. Confirm which "unavailable" fields are computable metrics versus external feeds versus new workflow capture.
  6. Pick 2-3 real-data slices for the first columnar store POC.
What we should say: We have mapped the use cases into the data fields needed to power them. Now we want to validate the source systems and integration paths with your team.
What we are not asking for yet: full production credentials, ERP changes, or a large integration program on day one.
What we are asking for: source owners, masked extracts, schemas, refresh expectations, and workshops where the data does not exist yet.
Best wedge: start with SAP BW / Power BI and internal-app use cases, build a governed columnar layer, then expand into EXIM, MCA/GST, credit, and logistics providers.
